# The Newport Maeve Chronicles

Official landing page for *The Newport Maeve Chronicles* — a gothic noir fantasy series by Dean Jordanov.

Live site: [newportmaeve.com](https://newportmaeve.com)

The site is a static, no-build website: plain HTML, CSS, and vanilla JavaScript (ES modules). Character content is data-driven from JSON, and the per-character pages are generated by a small Python script.

## How the character system works

All character data lives in **`data/characters.json`** — name, role, signature quote, profile fields (Name, Race, Abilities, Role, Age, Hair, Eyes, Affiliations, Traits), bio, and the image paths for each gallery slot.

The homepage reads this file at runtime to build the character cards (`js/components/characters.js`). Each card links to that character's page in `characters/`.

The pages in **`characters/` are generated** by `build_character_sheets.py` — they are not written by hand. Editing them directly will be overwritten the next time the script runs.

### Adding or updating a character's images

1. Save the images into that character's folder using the exact names, e.g. for Miroslav Dion:
   - `assets/images/miroslav-dion/miroslav-dion-portrait.webp`
   - `assets/images/miroslav-dion/miroslav-dion-item1.webp`
   - `assets/images/miroslav-dion/miroslav-dion-turnaround.webp`
2. That's it — the gallery slots already point at these paths. Until a file exists, the slot shows a styled "Coming soon" frame instead of a broken image (handled by an `onerror` fallback).

### Changing a character's text, labels, or adding a new slot

1. Edit the character's entry in `data/characters.json` (bio, profile fields, the third slot's label such as "Insignia" / "Sigil" / "Mark", etc.).
2. Regenerate the pages:
   ```bash
   python3 build_character_sheets.py
   ```
3. Commit the changed files in `characters/`.

> The third gallery slot's filename is `-item1` so additional small images (`-item2`, `-item3`, …) can be added later without renaming.

## SEO notes

- The homepage `BookSeries` JSON-LD is **inlined** in `index.html` (not loaded from `data/schema.json`). The old `data/schema.json` is unused and can be deleted.
- Each character page includes its own `<title>`, meta description, canonical URL, Open Graph / Twitter tags, and a `Person` JSON-LD linked to the series.
- `sitemap.xml` lists the homepage and every character page. If you add a character, add its URL here too.
- The two verification files at the repo root (`25e1fa98….txt`, `yandex_….html`) and `CNAME` must remain at the root for ownership checks and the custom domain to keep working.

## Local preview

No build step is required to view the site, but it must be served over HTTP (the `fetch()` calls and ES modules will not work from a `file://` path):

```bash
npx serve .
# or
python3 -m http.server 8000
```

Then open the printed URL (e.g. `http://localhost:8000`).

The only tooling step is regenerating character pages after editing `data/characters.json`:

```bash
python3 build_character_sheets.py
```

## Deployment

Hosted on **GitHub Pages** with a custom domain via `CNAME`.

1. Push to the branch configured for Pages (usually `main`).
2. GitHub serves files from the repository root.
3. The custom domain `newportmaeve.com` is set in **Settings → Pages**.

## Editing quick reference

| What to change | Where |
|----------------|-------|
| Homepage copy / sections | `index.html` |
| Character data (cards + sheets) | `data/characters.json`, then run `build_character_sheets.py` |
| Character images | `assets/images/<slug>/` (exact filenames above) |
| Site / social links | `data/site.json` |
| Homepage SEO / schema | `<head>` of `index.html` |
| Colors / fonts | `css/general.css`, `css/typography.css` |
| Sitemap | `sitemap.xml` |
